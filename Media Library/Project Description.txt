Необходимо составить библиотеку классов и интерфейсов для описания сущностей домена «Медиатека».

Image ================= фото
ImageURL ============== ссылка на фото (url) 
Track ================= муз.трек
TrackURL ============== ссылка на муз.трек
Video ================= видео
VideoURL ============== ссылка на видео
History =============== событие (набор из фото, ссылок на фото, видео, ссылок на видео) 
Disk ================== диск (фиксированный набор из треков и картинок)  ===================== класс контейнер
Serial ================ сериал (набор из видео + набор картинок) ============================= класс контейнер
Compilation =========== подборка (как диск только с возможностью изменения элементов и/или порядка элементов) === класс контейнер

/////////
Name - у всех
DateOfCreation - у всех
Rating - только у фильмов
Author - у всех
Genre - у всех
Duration - только аудио и видео
Id - у всех
AlbumArt - Disk, Serial, Compilation


Остальные требования:

1)     Составить демо

2)     Использовать шаблон Factory и/или Builder для построения сложных объектов

3)     Использовать принцип Dependency Injection

4)     В классах контейнерах обеспечить необходимые  CRUD операции (при необходимости) ==== create, read, update, delete

5)     Сделать методы для проигрывания элементов в некотором абстрактном плейере 
       (методы могут просто выводить в консоль название проигрываемого элемента)
